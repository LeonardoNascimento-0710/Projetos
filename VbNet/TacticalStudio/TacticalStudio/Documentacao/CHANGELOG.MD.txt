======================================================================
2. REFERÊNCIA VISUAL DO PROJETO
======================================================================

O projeto pretende chegar a uma prancheta tática com aparência semelhante
a um campo de treinamento de futebol, contendo:

- Campo com faixas de gramado;
- Jogadores movimentáveis;
- Jogadores de equipes diferentes;
- Goleiros;
- Bola;
- Cones;
- Gols;
- Manequins;
- Linhas contínuas;
- Linhas tracejadas;
- Áreas delimitadas;
- Numeração das etapas do exercício;
- Sistema de seleção e movimentação dos objetos;
- Controle de camadas;
- Futuro suporte a sprites.

======================================================================
3. DECISÃO DE ARQUITETURA
======================================================================

Foi definido que o FrmPrincipal não será responsável por desenhar e
movimentar diretamente os elementos do campo.

Responsabilidades definidas:

FrmPrincipal
- Controlar menus;
- Controlar painéis;
- Inicializar o campo tático;
- Gerenciar a interface principal;
- Acionar ferramentas e comandos.

CampoTatico
- Desenhar o gramado;
- Desenhar as marcações do campo;
- Desenhar os objetos táticos;
- Selecionar objetos;
- Movimentar objetos;
- Converter posições percentuais para coordenadas de tela;
- Impedir que os objetos saiam da área do campo.

TacticalStudio.Core
- Armazenar as classes de domínio;
- Representar jogadores, bola e futuros objetos;
- Manter propriedades independentes da interface gráfica.

4. ETAPA 1 - CENTRALIZAÇÃO DO CAMPO TÁTICO
======================================================================

STATUS: CONCLUÍDA

Arquivo alterado:
- Controles\CampoTatico.vb

Alterações realizadas:

- O CampoTatico passou a herdar UserControl;
- Foi ativado DoubleBuffered para evitar cintilação;
- Foi criado o desenho do gramado;
- Foram criadas faixas alternadas no gramado;
- Foram desenhadas as linhas externas;
- Foi desenhada a linha central;
- Foi desenhado o círculo central;
- Foi desenhado o ponto central;
- Foram desenhadas as grandes áreas;
- Foram desenhadas as pequenas áreas;
- Foi criado suporte inicial a jogadores;
- Os jogadores passaram a usar posição percentual;
- Foi implementada a seleção de jogadores;
- Foi implementado o arraste com o mouse;
- Foi criada uma borda dourada para indicar seleção;
- O jogador foi impedido de sair da área do campo.

Arquivo alterado:
- FrmPrincipal.vb

Alterações realizadas:

- Foi removida do formulário a responsabilidade de desenhar o campo;
- Foi criado o objeto CampoCanvas;
- O CampoTatico passou a ser inserido dentro do PnlCentral;
- O FrmPrincipal passou a inicializar o campo;
- Foi adicionado um jogador inicial para validação.

Arquivo alterado:
- TacticalStudio.Core\Classes\Jogador.vb

Alteração realizada:

- Foi removido um Imports incorreto ou desnecessário;
- A classe continuou dentro do namespace TacticalStudio.Core.Classes.

Resultado da etapa:

- O campo passou a funcionar como um componente independente;
- O jogador passou a ser selecionável e arrastável;
- A posição passou a permanecer proporcional ao redimensionamento;
- O FrmPrincipal ficou mais limpo.

5. ETAPA 2 - SISTEMA GENÉRICO DE OBJETOS E BOLA
======================================================================

STATUS: CONCLUÍDA

Arquivo criado:
- TacticalStudio.Core\Classes\Bola.vb

Estrutura:

- A classe Bola herda ObjetoCampo;
- A bola utiliza a posição e as propriedades comuns da classe base;
- A classe está preparada para receber novas propriedades futuramente.

Arquivo alterado:
- Controles\CampoTatico.vb

Alterações realizadas:

- A lista exclusiva de jogadores foi substituída por uma lista de
  ObjetoCampo;
- Jogadores e bolas passaram a compartilhar o mesmo sistema de seleção;
- Jogadores e bolas passaram a compartilhar o mesmo sistema de arraste;
- Foi criado o método AdicionarJogador;
- Foi criado o método AdicionarBola;
- Foi criado o método LimparObjetos;
- Foi criado o método DesenharObjetos;
- Foi criado o método DesenharJogador;
- Foi criado o método DesenharBola;
- Foi criado o método DesenharSelecao;
- Foi criado o método LocalizarObjetoNaPosicao;
- Foi criado o método MoverObjetoSelecionado;
- Foi criado o método DeselecionarTodos;
- Foi criada a identificação visual da bola;
- Foi criada uma sombra para a bola;
- Foi criada uma área de clique maior que o tamanho visual da bola;
- O cursor passa a mudar para Hand sobre objetos;
- Durante o arraste, o cursor passa a usar SizeAll;
- O sistema identifica primeiro o objeto desenhado por último;
- A movimentação passou a respeitar o tamanho visual de cada objeto.

Arquivo alterado:
- FrmPrincipal.vb

Alterações realizadas no método CriarCampoTatico:

- Foi mantida a criação do CampoCanvas;
- Foram adicionados dois jogadores de teste;
- Foi adicionada uma bola de teste;
- Os objetos foram posicionados por porcentagem.

Resultado da etapa:

- Dois jogadores podem ser selecionados e movimentados;
- A bola pode ser selecionada e movimentada;
- O mesmo sistema de movimentação poderá ser reutilizado por cones,
  gols, manequins e outros objetos;
- A estrutura ficou preparada para crescimento sem repetir lógica.

6. ESTRUTURA ATUAL DE OBJETOS
======================================================================

ObjetoCampo

- Deve concentrar propriedades comuns;
- Exemplos de propriedades:

  - Visivel;
  - Selecionado.

Jogador
- Herda ObjetoCampo;
- Possui número;
- Possui nome;
- Atualmente é desenhado como um círculo vermelho com número.

Bola
- Herda ObjetoCampo;
- Atualmente é desenhada diretamente com Graphics;
- Possui corpo branco, contorno e detalhes escuros.
