======================================================================

Este documento registra as alterações realizadas no projeto TacticalStudio.

- Arquivos criados;

======================================================================
2. REFERÊNCIA VISUAL DO PROJETO
======================================================================

O projeto pretende chegar a uma prancheta tática com aparência semelhante
a um campo de treinamento de futebol, contendo:

- Campo com faixas de gramado;
- Jogadores movimentáveis;
- Jogadores de equipes diferentes;
- Goleiros;
- Bola;
- Cones;
- Gols;
- Manequins;
- Linhas contínuas;
- Linhas tracejadas;
- Áreas delimitadas;
- Numeração das etapas do exercício;
- Sistema de seleção e movimentação dos objetos;
- Controle de camadas;
- Futuro suporte a sprites.

======================================================================
3. DECISÃO DE ARQUITETURA
======================================================================

Foi definido que o FrmPrincipal não será responsável por desenhar e
movimentar diretamente os elementos do campo.

Responsabilidades definidas:

FrmPrincipal
- Controlar menus;
- Controlar painéis;
- Inicializar o campo tático;
- Gerenciar a interface principal;
- Acionar ferramentas e comandos.

CampoTatico
- Desenhar o gramado;
- Desenhar as marcações do campo;
- Desenhar os objetos táticos;
- Selecionar objetos;
- Movimentar objetos;
- Converter posições percentuais para coordenadas de tela;
- Impedir que os objetos saiam da área do campo.

TacticalStudio.Core
- Armazenar as classes de domínio;
- Representar jogadores, bola e futuros objetos;
- Manter propriedades independentes da interface gráfica.
